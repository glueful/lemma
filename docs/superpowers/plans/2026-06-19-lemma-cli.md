# `lemma` CLI & Onboarding (v1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a branded `lemma` operator CLI (`lemma setup` / `lemma doctor` + a thin bin wrapper) that takes an existing Lemma checkout from "DB credentials in hand" to "first admin exists, site configured" — Layer 1 via the framework's `Glueful\Installer\` seams, Layer 2 via the shared `App\Setup\SetupService`.

**Architecture:** Two `php glueful` console commands (`lemma:setup`, `lemma:doctor`) in a new top-level `App\Setup\` module, fronted by a POSIX-`sh` `lemma` launcher. The commands are thin: they gather input (prompts when interactive, env/options when not) and delegate to small, unit-tested seams — `PgsqlDatabaseConfigFactory` (input → `DatabaseConfig`), `Doctor` (environment checks), and `SetupRunner` (orchestrates `InstallerRunner` + `SetupService`). The framework `Installer` performs the single DB connection test (preflight) and the migration, so "tested == migrated" holds without us re-implementing it.

**Tech Stack:** PHP 8.3, Glueful console (`Glueful\Console\BaseCommand`, Symfony Console under the hood), the framework `Glueful\Installer\` seams (`Installer`, `InstallOptions`, `DatabaseConfig`, `ConnectionTester`, `InstallResult`/`InstallStep`, `EnvWriter`), PHPUnit 10.

**Design spec:** `docs/superpowers/specs/2026-06-19-lemma-cli-onboarding-design.md`

#!/usr/bin/env sh
# lemma — branded operator front door for a Lemma instance.
# A thin developer-convenience launcher: it forwards to the app's own `php glueful`
# console (which owns the actual lemma:setup / lemma:doctor commands). It contains NO
# setup logic of its own.
set -e

DIR="$(cd "$(dirname "$0")" && pwd)"
PHP="${PHP:-php}"
GLUEFUL="$DIR/glueful"

cmd="${1:-}"
[ "$#" -gt 0 ] && shift || true

# Every word is quoted so a $DIR (or any arg) containing spaces is passed intact.
case "$cmd" in
  "" | -h | --help | help)
    exec "$PHP" "$GLUEFUL" list
    ;;
  migrate)
    exec "$PHP" "$GLUEFUL" migrate:run "$@"
    ;;
  key:generate)
    exec "$PHP" "$GLUEFUL" generate:key "$@"
    ;;
  *)
    # setup, doctor, and any other lemma:<cmd>
    exec "$PHP" "$GLUEFUL" "lemma:$cmd" "$@"
    ;;
esac
```

<?php

declare(strict_types=1);

namespace App\Setup;

use Glueful\Installer\DatabaseConfig;
use Glueful\Installer\EnvWriter;

/**
 * Builds a Postgres DatabaseConfig for first-run setup. Lemma is Postgres-required, so the engine
 * is always 'pgsql' — never prompted. This is the single place that knows Lemma's DB_PGSQL_* env
 * key names; both interactive and --quiet (from-env) input funnel through here.
 */
final class PgsqlDatabaseConfigFactory
{
    public function fromInput(
        string $host,
        int $port,
        string $database,
        string $username,
        string $password,
        ?string $schema = null,
        ?string $sslMode = null,
    ): DatabaseConfig {
        return new DatabaseConfig('pgsql', $host, $port, $database, $username, $password, $schema, $sslMode);
    }

    public function fromEnv(EnvWriter $env): DatabaseConfig
    {
        $schema = $env->get('DB_PGSQL_SCHEMA');
        $sslMode = $env->get('DB_PGSQL_SSL_MODE');

        return new DatabaseConfig(
            'pgsql',
            $env->get('DB_PGSQL_HOST') ?? '',
            $this->parsePort($env->get('DB_PGSQL_PORT')),
            $env->get('DB_PGSQL_DATABASE') ?? '',
            $env->get('DB_PGSQL_USERNAME') ?? '',
            $env->get('DB_PGSQL_PASSWORD') ?? '',
            ($schema === null || $schema === '') ? null : $schema,
            ($sslMode === null || $sslMode === '') ? null : $sslMode,
        );
    }

    /**
     * Absent => the pgsql default (5432). A non-numeric value (e.g. "5432abc") becomes 0 so it is
     * caught by requiredFieldErrors() rather than silently truncated by an (int) cast.
     */
    private function parsePort(?string $raw): int
    {
        return match (true) {
            $raw === null || $raw === '' => 5432,
            ctype_digit($raw) => (int) $raw,
            default => 0,
        };
    }

    /**
     * The required-field names that are missing or invalid in $config. Empty list => valid.
     * Used to FAIL LOUDLY before the Installer runs — both in --quiet mode (env may be blank/partial)
     * and interactively (a prompt may have been answered empty).
     *
     * @return list<string>
     */
    public function requiredFieldErrors(DatabaseConfig $config): array
    {
        $errors = [];
        if ($config->host === '') {
            $errors[] = 'host (DB_PGSQL_HOST)';
        }
        if ($config->port <= 0) {
            $errors[] = 'port (DB_PGSQL_PORT)';
        }
        if ($config->database === '') {
            $errors[] = 'database (DB_PGSQL_DATABASE)';
        }
        if ($config->username === '') {
            $errors[] = 'username (DB_PGSQL_USERNAME)';
        }
        if ($config->password === '') {
            $errors[] = 'password (DB_PGSQL_PASSWORD)';
        }
        return $errors;
    }
}
```

declare(strict_types=1);

namespace App\Setup\Doctor;

/** One environment check's result. */
final class Check
{
    public const OK = 'ok';
    public const WARN = 'warning';
    public const FAIL = 'fail';

    public function __construct(
        public readonly string $name,
        public readonly string $status,
        public readonly string $message,
    ) {
    }

    public static function ok(string $name, string $message): self
    {
        return new self($name, self::OK, $message);
    }

    public static function warn(string $name, string $message): self
    {
        return new self($name, self::WARN, $message);
    }

    public static function fail(string $name, string $message): self
    {
        return new self($name, self::FAIL, $message);
    }
}

 * First-run environment checks. Two phases:
 *  - preflight(): no DB credentials needed; runs BEFORE any prompt.
 *  - reachability(): needs a DatabaseConfig; runs after creds exist (or against existing env).
 *
 * Runtime facts (PHP version, loaded extensions, base path) are injected so every branch is
 * unit-testable. `lemma:setup` calls preflight() before prompting; `lemma:doctor` calls both.
 */
final class Doctor
{
    private const MIN_PHP = '8.3.0';
    private const REQUIRED_EXTENSIONS = ['pdo_pgsql'];

    /** @param list<string> $loadedExtensions */
    public function __construct(
        private readonly string $basePath,
        private readonly string $phpVersion,
        private readonly array $loadedExtensions,
    ) {
    }

    /** @return list<Check> */
    public function preflight(): array
    {
        $checks = [];

        $checks[] = version_compare($this->phpVersion, self::MIN_PHP, '>=')
            ? Check::ok('php', "PHP {$this->phpVersion} (>= " . self::MIN_PHP . ').')
            : Check::fail('php', "PHP {$this->phpVersion} is below the required " . self::MIN_PHP . '.');

        foreach (self::REQUIRED_EXTENSIONS as $ext) {
            $checks[] = in_array($ext, $this->loadedExtensions, true)
                ? Check::ok("ext:{$ext}", "Extension {$ext} loaded.")
                : Check::fail("ext:{$ext}", "Required PHP extension {$ext} is not loaded.");
        }

        $checks[] = $this->envTargetCheck();
        $checks[] = $this->writableStorageCheck();
        $checks[] = $this->keysCheck();

        return $checks;
    }

    public function reachability(DatabaseConfig $config, ConnectionTester $tester): Check
    {
        $result = $tester->test($config);

        return $result->ok
            ? Check::ok('database', "Connected to {$config->host}:{$config->port}/{$config->database}.")
            : Check::fail('database', $result->message);
    }

    /**
     * The `.env` we will WRITE must be reachable: if it exists it must be writable; if it does not
     * exist, the project root must be writable AND .env.example must be readable (the framework
     * Installer creates .env from .env.example). "keys present" is intentionally NOT checked here —
     * Layer 1 generates them.
     */
    private function envTargetCheck(): Check
    {
        $env = $this->basePath . '/.env';
        if (is_file($env)) {
            return is_writable($env)
                ? Check::ok('env-target', '.env is writable.')
                : Check::fail('env-target', '.env exists but is not writable.');
        }

        $example = $this->basePath . '/.env.example';
        if (!is_file($example) || !is_readable($example)) {
            return Check::fail('env-target', '.env is absent and .env.example is missing/unreadable.');
        }
        if (!is_writable($this->basePath)) {
            return Check::fail('env-target', 'Project root is not writable; cannot create .env.');
        }

        return Check::ok('env-target', '.env will be created from .env.example.');
    }

    private function writableStorageCheck(): Check
    {
        $storage = $this->basePath . '/storage';
        return is_dir($storage) && is_writable($storage)
            ? Check::ok('storage', 'storage/ is writable.')
            : Check::fail('storage', 'storage/ is missing or not writable.');
    }

    /**
     * Security keys are a WARNING when absent, never a hard preflight FAIL — on a fresh checkout
     * they are absent BY DESIGN and `lemma:setup` generates them via the Installer. Standalone
     * `lemma:doctor --strict` promotes this WARN to a failure (a configured-but-keyless install is
     * a real post-setup problem). All three framework keys are checked (TOKEN_SALT included).
     */
    private function keysCheck(): Check
    {
        $env = $this->basePath . '/.env';
        if (!is_file($env)) {
            return Check::warn('keys', 'No .env yet — security keys will be generated by setup.');
        }

        $writer = new EnvWriter($env);
        $missing = [];
        foreach (['APP_KEY', 'TOKEN_SALT', 'JWT_KEY'] as $key) {
            if ((string) ($writer->get($key) ?? '') === '') {
                $missing[] = $key;
            }
        }

        return $missing === []
            ? Check::ok('keys', 'Security keys present.')
            : Check::warn('keys', 'Missing security keys: ' . implode(', ', $missing) . '.');
    }
}

use App\Setup\Doctor\Check;

    name: 'lemma:doctor',
    description: 'Check that this host can run a Lemma instance (PHP, extensions, paths, database)',
)]

    protected function configure(): void
    {
        $this->addOption(
            'strict',
            null,
            InputOption::VALUE_NONE,
            'Treat warnings (e.g. absent security keys) as failures',
        );
    }

    protected function execute(InputInterface $input, OutputInterface $output): int
    {

        $basePath = base_path($this->getContext());

        $checks = $doctor->preflight();

        // Reachability only when the DB is already configured in .env (best-effort).

        if ($state->isDatabaseConfigured()) {
            $config = (new PgsqlDatabaseConfigFactory())->fromEnv(new EnvWriter($basePath . '/.env'));
            $checks[] = $doctor->reachability($config, new ConnectionTester($this->getContext()));
        }

        $rows = [];
        $failed = false;
        foreach ($checks as $check) {
            $rows[] = [$this->badge($check->status), $check->name, $check->message];
            // A FAIL always fails; under --strict a WARN (e.g. missing keys) fails too.
            $failed = $failed
                || $check->status === Check::FAIL
                || ($strict && $check->status === Check::WARN);
        }
        $this->table(['', 'Check', 'Detail'], $rows);

        if ($failed) {
            $this->error('Some checks failed. Resolve them, then run `lemma setup`.');
            return self::FAILURE;
        }

        $this->success('Environment looks healthy.');
        return self::SUCCESS;
    }

    private function badge(string $status): string
    {
        return match ($status) {
            Check::OK => 'OK',
            Check::WARN => 'WARN',
            default => 'FAIL',
        };
    }
}

```php
<?php

declare(strict_types=1);

namespace App\Setup;

use App\Setup\Contracts\InstallerRunner;
use App\Setup\Contracts\SetupInstaller;
use Glueful\Installer\InstallOptions;

/**
 * Orchestrates a `lemma:setup` run:
 *   Layer 1 — the framework Installer (preflight → .env → keys → migrate), via InstallerRunner.
 *   Layer 2 — first admin + site settings, via SetupInstaller (App\Setup\SetupService).
 *
 * Policy:
 *  - If Layer 1 fails, Layer 2 never runs (nothing was migrated).
 *  - Layer 2 is PERMANENT: once installed, the first admin is never re-created. `--force` flows to
 *    Layer 1 only (regenerate keys / rewrite .env / re-run pending migrations); it does NOT bypass
 *    the Layer-2 lock — SetupService is the permanent, race-safe lock by design. So when already
 *    installed, Layer 2 is skipped regardless of --force.
 */
final class SetupRunner
{
    public function __construct(
        private readonly InstallerRunner $installer,
        private readonly SetupInstaller $setup,
    ) {
    }

    public function run(SetupRequest $request): SetupOutcome
    {
        // --force flows to Layer 1 here (keys/.env/migrate); it never re-creates the admin.
        $result = $this->installer->run(new InstallOptions(
            database: $request->database,
            skipDatabase: $request->skipDatabase,
            force: $request->force,
        ));

        if (!$result->ok) {
            return new SetupOutcome(false, $result->steps, false, false, $request);
        }

        // No DB in scope => no Layer 2 (the admin lives in the DB).
        if ($request->skipDatabase) {
            return new SetupOutcome(true, $result->steps, false, false, $request);
        }

        // Layer 2 is one-shot and permanent — already installed means skip, even under --force.
        if ($this->setup->isInstalled()) {
            return new SetupOutcome(true, $result->steps, false, true, $request);
        }

        $this->setup->install(
            $request->siteName,
            $request->adminEmail,
            $request->adminPassword,
            $request->locale,
        );

        return new SetupOutcome(true, $result->steps, true, false, $request);
    }
}
```

```php
<?php

declare(strict_types=1);

namespace App\Setup\Console;

use App\Setup\Doctor\Check;
use App\Setup\Doctor\Doctor;
use App\Setup\PgsqlDatabaseConfigFactory;
use App\Setup\SetupRequest;
use App\Setup\SetupRunner;
use Glueful\Console\BaseCommand;
use Glueful\Installer\DatabaseConfig;
use Glueful\Installer\EnvWriter;
use Symfony\Component\Console\Attribute\AsCommand;
use Symfony\Component\Console\Input\InputInterface;
use Symfony\Component\Console\Input\InputOption;
use Symfony\Component\Console\Output\OutputInterface;

use function base_path;

#[AsCommand(
    name: 'lemma:setup',
    description: 'Configure the database and create the first admin for this Lemma instance',
)]
final class SetupCommand extends BaseCommand
{
    protected function configure(): void
    {
        $this->addOption('force', null, InputOption::VALUE_NONE, 'Re-run past the installed guard');
        $this->addOption('quiet', null, InputOption::VALUE_NONE, 'Non-interactive: read DB creds from .env, take Layer-2 values from options');
        $this->addOption('skip-database', null, InputOption::VALUE_NONE, 'Skip DB setup and migrations (no admin is created)');
        // Layer-2 values for --quiet mode:
        $this->addOption('admin-email', null, InputOption::VALUE_REQUIRED, 'First admin email (quiet mode)');
        $this->addOption('admin-password', null, InputOption::VALUE_REQUIRED, 'First admin password (quiet mode)');
        $this->addOption('site-name', null, InputOption::VALUE_REQUIRED, 'Site name (quiet mode)', 'Lemma');
        $this->addOption('locale', null, InputOption::VALUE_REQUIRED, 'Default locale (quiet mode)', 'en');
    }

    protected function execute(InputInterface $input, OutputInterface $output): int
    {
        $basePath = base_path($this->getContext());
        $quiet = (bool) $input->getOption('quiet') || !$this->isInteractive();
        $force = (bool) $input->getOption('force');
        $skipDb = (bool) $input->getOption('skip-database');

        $factory = new PgsqlDatabaseConfigFactory();

        // 1. Validate required quiet-mode inputs FIRST — fail loudly before any side effect.
        //    Contract: --skip-database means "no Layer 2", so admin options are NOT required then.
        if ($quiet && !$skipDb) {
            foreach (['admin-email', 'admin-password'] as $required) {
                if ((string) ($input->getOption($required) ?? '') === '') {
                    $this->error("Missing required --{$required} in non-interactive mode.");
                    return self::FAILURE;
                }
            }
        }

        // 2. Pre-prompt environment checks (no credentials needed).
        $doctor = new Doctor($basePath, PHP_VERSION, get_loaded_extensions());
        foreach ($doctor->preflight() as $check) {
            if ($check->status === Check::FAIL) {
                $this->error("Preflight failed ({$check->name}): {$check->message}");
                return self::FAILURE;
            }
        }

        // 3. Resolve the database config (skipped entirely with --skip-database), then validate it
        //    so a blank/partial .env (quiet) or an empty prompt fails LOUDLY before the Installer.
        $database = null;
        if (!$skipDb) {
            $database = $quiet
                ? $factory->fromEnv(new EnvWriter($basePath . '/.env'))
                : $this->promptDatabase($factory);

            $missing = $factory->requiredFieldErrors($database);
            if ($missing !== []) {
                $this->error(
                    'Missing/invalid database settings: ' . implode(', ', $missing)
                    . ($quiet ? ' — set DB_PGSQL_* in .env.' : '.'),
                );
                return self::FAILURE;
            }
        }

        // 4. Resolve Layer-2 values.
        [$siteName, $adminEmail, $adminPassword, $locale] = $quiet
            ? [
                (string) $input->getOption('site-name'),
                (string) $input->getOption('admin-email'),
                (string) $input->getOption('admin-password'),
                (string) $input->getOption('locale'),
            ]
            : $this->promptAdmin();

        // 5. Run both layers.
        $request = new SetupRequest($database, $siteName, $adminEmail, $adminPassword, $locale, $force, $skipDb);
        /** @var SetupRunner $runner */
        $runner = $this->getService(SetupRunner::class);
        $outcome = $runner->run($request);

        // 6. Render.
        $this->table(['Step', 'Status', 'Detail'], array_map(
            static fn ($s) => [$s->name, $s->status, $s->message],
            $outcome->steps,
        ));

        if (!$outcome->ok) {
            $this->error('Setup failed: ' . $outcome->firstFailureMessage());
            return self::FAILURE;
        }

        if ($outcome->alreadyInstalled) {
            $this->info($force
                ? 'Already installed — Layer 1 infrastructure was refreshed; the first admin is preserved (never re-created).'
                : 'Already installed — the first admin already exists; nothing to do for Layer 2.');
        }

        $this->renderSummary($outcome);
        return self::SUCCESS;
    }

    /** @return array{0:string,1:string,2:string,3:string} site, email, password, locale */
    private function promptAdmin(): array
    {
        $siteName = $this->ask('Site name', 'Lemma');
        $adminEmail = $this->ask('First admin email');
        $adminPassword = $this->secret('First admin password (min 12 chars)');
        $locale = $this->ask('Default locale', 'en');

        return [$siteName, $adminEmail, $adminPassword, $locale];
    }

    private function promptDatabase(PgsqlDatabaseConfigFactory $factory): DatabaseConfig
    {
        $host = $this->ask('Postgres host', 'localhost');
        $port = (int) $this->ask('Postgres port', '5432');
        $database = $this->ask('Database name');
        $username = $this->ask('Database user');
        $password = $this->secret('Database password');
        $schema = $this->ask('Schema', 'public');
        $sslMode = $this->ask('SSL mode (disable/prefer/require)', 'prefer');

        return $factory->fromInput($host, $port, $database, $username, $password, $schema, $sslMode);
    }

    private function renderSummary(\App\Setup\SetupOutcome $outcome): void
    {
        $req = $outcome->request;
        if ($req->database !== null) {
            // Postgres is fixed; the password is never shown.
            $this->info(sprintf(
                'Database: %s:%d/%s (migrations applied).',
                $req->database->host,
                $req->database->port,
                $req->database->database,
            ));
        }
        if ($outcome->adminCreated) {
            $baseUrl = (string) config($this->getContext(), 'app.urls.base', 'http://localhost');
            $this->success(sprintf('Admin %s created. Sign in at %s/admin', $req->adminEmail, rtrim($baseUrl, '/')));
        }
        $this->note('Next: start the app (`php glueful serve`) and sign in to the admin.');
    }
}
```
